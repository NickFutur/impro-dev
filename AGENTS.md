# Правила работы с проектом

Проект — HTML/SCSS/JS/Gulp-верстка, которую в будущем могут переносить на 1С-Битрикс.

## Главные правила

1. Не ломать текущую визуальную логику.
2. Не менять классы без необходимости.
3. Соблюдать БЭМ-подход.
4. Не добавлять тяжелые зависимости без причины.
5. Не переписывать весь файл, если можно исправить локально.
6. Перед изменениями понять структуру проекта.
7. После изменений запускать проверки.

## Проверки

Перед финальным ответом выполнить:

- npm run check
- npm run check повторно

## HTML

- Использовать семантичные теги.
- Следить за alt у изображений.
- Не дублировать лишние элементы ради визуала, если можно решить SCSS.
- Не использовать inline-style без необходимости.

## SCSS

- Не плодить хаотичные селекторы.
- Использовать вложенность аккуратно.
- Не превышать 3–4 уровня вложенности.
- Общие значения выносить в переменные/mixins, если они повторяются.
- Не ломать существующую адаптивность.

## JS

- Не использовать глобальные переменные без необходимости.
- Проверять существование элементов перед обработчиками.
- Не ломать текущую логику Gulp-сборки.

## Подготовка под 1С-Битрикс

- Верстка должна быть модульной.
- Блоки должны быть переносимыми.
- Не завязывать логику на случайную структуру страницы.
- Компоненты должны быть понятны другому разработчику.
- готовить код под Bitrix
- не ломать адаптив

## Bitrix Skills

В проекте подключён набор skills:

skills/bitrix-skills/
Перед задачами по подготовке вёрстки к 1С-Битрикс нужно использовать:

skills/bitrix-skills/AGENTS.md
skills/bitrix-skills/.agents/skills/
skills/bitrix-skills/shared/
Если задача связана с Bitrix-ready аудитом, семантикой HTML, повторяющимися блоками, будущими компонентами, инфоблоками, include areas или безопасной подготовкой вёрстки к посадке, сначала прочитать соответствующий SKILL.md.

Правила:

Не переводить проект на PHP без отдельной команды.
Не ломать GitHub Pages preview.
Сначала составлять отчёт.
Только потом предлагать безопасные изменения.
Изменения с риском medium и high не выполнять без подтверждения.


## Как запускать skill в Codex

Команды ниже — это не команды терминала.

Это команды-промпты, которые нужно писать в Codex в чате задачи.

Например:

/audit-layout

Используй skill:
skills/bitrix-skills/.agents/skills/bitrix-layout-auditor/SKILL.md

Задача:
проведи аудит текущей HTML/SCSS/JS-вёрстки.
Код пока не менять.
Сначала выдай отчёт.
